GETTING STARTED
═══════════════════════════════════════════════════════════════════════════

Windows (PowerShell):
  1. cd "d:\projet erp\Location_voiture_web\location_voiture"
  2. .\setup.bat
  3. python manage.py runserver
  4. Visit http://localhost:8000

Linux/Mac (Terminal):
  1. cd location_voiture
  2. bash setup.sh
  3. python manage.py runserver
  4. Visit http://localhost:8000

Manual Setup:
  1. Create virtual environment: python -m venv venv
  2. Activate: source venv/bin/activate
  3. Install: pip install -r requirements.txt
  4. Migrate: python manage.py migrate
  5. Create superuser: python manage.py createsuperuser
  6. Run: python manage.py runserver

ODOO INTEGRATION SETUP
═══════════════════════════════════════════════════════════════════════════

1. Update ODOO_CONFIG in location_voiture/settings.py:
   - BASE_URL: Your Odoo server URL
   - DATABASE: Your Odoo database name
   - USERNAME: Odoo admin username
   - PASSWORD: Odoo admin password
   - USE_XMLRPC: True for XML-RPC, False for REST API

2. Test connection:
   python manage.py shell
   >>> from odoo_integration import OdooConnector
   >>> connector = OdooConnector()
   >>> connector.test_connection()

3. Sync cars:
   python manage.py sync_cars_from_odoo

4. Use in code:
   from odoo_integration import OdooConnector, SyncManager
   connector = OdooConnector()
   cars = connector.fetch_cars()

TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════

Issue: "ModuleNotFoundError: No module named 'django'"
  → Run: pip install -r requirements.txt

Issue: "DatabaseError" on first run
  → Run: python manage.py migrate

Issue: Static files not loading
  → Run: python manage.py collectstatic --noinput

Issue: Odoo connection fails
  → Check Odoo server is running
  → Verify credentials in settings.py
  → Check database name is correct
